Name

git-remote - manages the set of remotes that you are tracking with your local repository

SYNOPSIS

git remote [-v | --verbose]
git remote add [-t ] [-m ] [-f] [--[no-]tags] [--mirror=(fetch|push)]
git remote rename [--[no-]progress]
git remote remove
git remote set-head (-a | --auto | -d | --delete | )
git remote set-branches [--add] …​
git remote get-url [--push] [--all]
git remote set-url [--push] []
git remote set-url --add [--push]
git remote set-url --delete [--push]
git remote [-v | --verbose] show [-n] …​
git remote prune [-n | --dry-run] …​
git remote [-v | --verbose] update [-p | --prune] [( | )…​]

DESCRIPTION

This command manages the set of remotes that you are tracking with your local repository. If you try running git remote -v in your repositories, you'll probably see something called origin. You may notice origin in many messages from Git. origin is the human-friendly name for the URL that the remote repository is stored at. It's like a key value pair, and origin is the default.

OPTIONS

-v
